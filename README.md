# パチンコ実戦シミュレーター

実在2機種のスペックを忠実に再現した「数字だけ」のパチンコ実戦シミュレーションWebアプリ。
玉の物理挙動・液晶演出は再現せず、回転数・大当り・出玉・収支を数値とグラフで表現する。

サーバー・DB・外部APIは一切不要。すべてブラウザ内（localStorage）で完結する。

遊戯画面では回転速度（5/10/20/50回転/秒）を遊戯中でもボタン一つで変更できる。また
スランプグラフに加えて、通常時から何回転で当たり・何連チャンしたかを棒グラフで表示する
「大当り履歴」パネルも表示される（2連以上はオレンジ色＋「N連」バッジで強調）。

## 起動方法

ビルド不要。`index.html` をブラウザ（Chrome / Edge / Firefox 推奨）で直接開くだけで動作する。

```
pachinko-simulator/index.html をダブルクリック、またはブラウザにドラッグ&ドロップ
```

ローカルサーバーを使いたい場合は任意の静的サーバーで配信してもよい（例: `npx serve .`）。

状態（所持金・持ち玉・進行中の遊戯セッション）は `localStorage` に自動保存され、
ページをリロードしても続きから遊べる。「所持金・記録をリセットする」ボタンで初期状態に戻せる。

## ディレクトリ構成

```
pachinko-simulator/
├── index.html          エントリーHTML（5画面のSPA）
├── css/style.css        データランプ風UIのスタイル
├── js/
│   ├── config.js          貸玉単価・換金率・回転効率などの設定値
│   ├── machines.js        機種マスタ（確率・振り分け・出玉。出典は下記）
│   ├── rng.js              抽選ロジック（純粋関数）＋検証用シミュレーション関数
│   ├── state.js             状態管理（単一の状態オブジェクト＋localStorage永続化）
│   ├── gameEngine.js         回転ループ・投資・精算などの操作ロジック
│   └── ui/
│       ├── counter.js          データカウンター表示・大当り演出
│       ├── graph.js             スランプグラフ（Canvas）
│       └── screens.js           画面切替・イベント配線
├── assets/
│   ├── eva.svg / sao.svg   オリジナルの抽象化した筐体イラスト（版権キャラクターの模写ではない）
└── tests/
    └── verify.js            Node.jsで実行する大量回転シミュレーション検証スクリプト
```

技術スタックは素のHTML/CSS/JavaScript（ビルド不要）。`<script type="module">` は
`file://` で開いた際にブラウザのCORS制限で読み込めない場合があるため使用せず、
全スクリプトを通常の `<script>` として読み込み順（config → machines → rng → state →
gameEngine → ui/* → main）を守ってグローバルスコープで連携させている。

## 抽選ロジックの検証

```
node tests/verify.js
```

エヴァ・SAOそれぞれ200万回転のシミュレーションを行い、通常時大当り確率・初当り振り分け・
時短引き戻し率・ST/RUSH継続率などが仕様値に近いことを確認できる。

ブラウザのコンソールからも `runSimulation('eva', 100000)` / `runSimulation('sao', 100000)`
を直接呼び出して検証できる（`rng.js` がグローバルに公開している）。

## 機種マスタの数値・出典

`js/machines.js` に定数として集約。数値は本アプリの開発指示プロンプトに記載された
実機公表スペック（DMMぱちタウン調べ）に基づく。

- **e 新世紀エヴァンゲリオン 〜はじまりの記憶〜**（ビスティ）
  通常 1/399.9・ST中 1/99.6、初当り振り分け ST直行50.5%（うち10R・約1,500個が0.5%、
  2R・約300個が50%）／時短49.5%（2R・約300個）、時短100回転、ST157回転（継続率約80%）、
  トータルST突入率約61.4%、ST中の当りはオール2,400個
- **e ソードアート・オンライン 閃光の軌跡**（京楽）
  通常 1/199.9・RUSH中実質1/59.9、初当り振り分け SWORD RUSH 69%（2R・約300個）／
  LIGHTNING RUSH直行1%（10R・約1,500個）／通常復帰30%（2R・約300個）、
  SWORD RUSH実質54回転（継続率約70%、うち30%はLIGHTNING RUSHへ昇格・常に約1,500個、
  継続時は10R・約1,500個40%／2R・約300個30%）、LIGHTNING RUSH実質119回転
  （継続率約70%、10R・約1,500個70%／2R・約300個30%）

## お金と玉の管理ルール

- 初期所持金 1,000,000円／初期持ち玉 0玉
- 貸玉 4.3円/玉（1,000円=232玉）、換金 3.5円/玉（端数切り捨て）
- 回転効率 17回転/1,000円 → 1回転あたりの消費玉数は 232÷17 ≒ 13.65玉
- ST中（エヴァのST／SAOのSWORD RUSH・LIGHTNING RUSH）および時短中は持ち玉が減らない

これらの数値はすべて `js/config.js` に集約されており、変更するだけで挙動を調整できる。
